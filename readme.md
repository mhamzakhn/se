ZAAN is a full-stack food ordering website built for a university campus setting. Anyone can sign up and place orders, but users with a LUMS email (@lums.edu.pk) get a special discount, making it a perk for the LUMS community.
Users create an account with their email, verify it via a one-time password sent to their inbox, and then log in to get a JWT token that authenticates all their requests. The menu page shows available food items filtered by category (desi, chinese, beverages, etc.) with prices and images. Users can add items to their cart, adjust quantities, and checkout with special instructions. The cart is stored per-user in the database so it persists across sessions. There's also a forgot password flow that uses email OTP verification to let users reset their credentials.
On the admin side, authorized users can manage the full menu (add, edit, delete items), view incoming orders and update their status (preparing, ready, delivered), and send broadcast emails to all registered users for announcements.
The backend runs on Node.js with Express and uses MongoDB for all persistent data (users, menu items, carts, orders) and Redis for temporary OTP storage during signup and password reset. Authentication uses JWT tokens with two middleware layers — one to verify the token and another to check admin role. Emails are sent through Nodemailer using Gmail SMTP.
The frontend is a React SPA built with Vite and Tailwind CSS, using React Router for navigation and React Context for managing auth and cart state globally. The whole project is a monorepo with separate frontend and backend folders. For deployment, the backend runs as a Vercel serverless function and the frontend deploys as a static Vite build, with MongoDB Atlas and Upstash Redis replacing local instances in production.
